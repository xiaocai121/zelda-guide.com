# 塞尔达攻略网站 · UIUX 设计调研文档

**设计总监**：颜好看 ｜ **版本**：v0.1（MVP 设计调研） ｜ **日期**：2026-07-15
**配套文档**：`ZeldaGuide_PRD.md`（PM 许清楚，v0.1）
**设计哲学**：Anti-slop 级——禁止紫色渐变、禁止 emoji 图标、禁止千篇一律的 Hero 大图；产出"看不出是 AI 做的"精美 UI，对标 Linear / Stripe / Notion / Fandom 的设计完成度。

## 0. 假设说明（推进前提，未向用户确认，按合理假设推进）

- **D1 产品定位**：内容型 / 编辑型攻略站，读者是"来查攻略的人"，不是"来逛社区的人"。设计优先级 = 一眼找到 > 完整 > 好读 > 美观。
- **D2 双主题**：浅色为默认（长文阅读友好、百度/Google 首屏信任感强）；深色作为"夜间模式"开关（玩家常深夜游戏）。两套 Token 同步交付。
- **D3 广告**：尊重 PRD 已确认的 4 个广告位（顶部信息流 / 文末 / 侧边 sticky / 地图 overlay），但 UX 上强制"原生、可识别、不阻断阅读"，这是本设计文档的核心约束。
- **D4 图标系统**：全局唯一图标库 **Lucide**，尺寸 16/20/24px；**绝对禁止 emoji** 充当功能图标。
- **D5 字体系统**：`Inter` + `Noto Sans SC`（中文正文/UI）+ `JetBrains Mono`（数据表/坐标/配方比例）。
- **D6 反模式红线**：紫色渐变、emoji 图标、三列功能卡、千篇一律"大标题+副标题+抽象 3D"Hero 一律禁用。
- **D7 市场切换**：国内 / 海外按 `locale` 自动判定，广告联盟按 IP 切换（沿用 PRD A4），UI 布局与文案双市场一致。
- **D8 调研局限**：Game8 与 Fandom Zelda Wiki 在抓取时被 Cloudflare 拦截（返回 403 / "Just a moment..." 验证页），其信息架构与广告特征基于公开已知结构 + 合理假设补全，已在文中标注「假设」。Zelda Dungeon 与游民星空为实测抓取。

## 1. 竞品 UI 调研（信息架构 + 视觉问题 + 广告位）

### 1.1 Zelda Dungeon（zeldadungeon.net）— 实测

- **信息架构**：顶部全局导航（Games 下拉含全系列；Guides / Wiki / Maps / Videos / Music）。首页是**新闻编辑流**——实测首页顶部满是 "Daily Debate / Fan Fiction Friday / The Zelda Cast Episode 428" 等社媒式内容（抓取可见），对"找攻略"的玩家噪声极大。**攻略/维基页**才是正题：左侧分类树（按游戏 → Walkthrough / Shrines / Side Quests / Collectibles / Bosses / Items），主内容区 + 右栏。
- **视觉风格**：历史深绿/深色调，密度中等，字体偏传统（非 Inter 级），整体"粉丝站"气质。
- **广告位**：几乎无（其优势 = 体验干净；其弱点 = 变现弱，不是我们的路线）。
- **借鉴**：① 左侧分类树清晰；② 互动地图体验好。
- **规避**：① 首页沦为"新闻编辑部"（我们要攻略优先）；② TOTK 攻略"至今未做完"（内容完整度靠自动补抓兜底，见 PRD）。

### 1.2 Game8（game8.co）— 假设（Cloudflare 拦截）

- **信息架构**：游戏落地页 + 超大 Tab 导航切游戏（TOTK / BOTW …）；**左侧 sticky 分类菜单**（攻略 / 地图 / 数据 / 角色 / 攻略大全），主内容区，**右栏"跳到章节" TOC + 相关链接**。结构化极强、移动优先、SEO 极强（PRD 记其 Google SEO 极强）。
- **视觉风格**：干净、信息密度高、卡片化、略模板化。
- **广告位**：**密集**——顶部 leaderboard、文中原生广告、右侧 sticky、移动端底部 sticky；Premium 文章需会员去广告。
- **借鉴**：① "左分类 + 右 TOC"的**双栏锚定结构**，极适合长攻略；② 结构化数据/SEO 实践。
- **规避**：① 广告过密（我们要克制）；② 内容模板化偏浅（我们要原创深度打法）。

### 1.3 Fandom Zelda Wiki（zelda.fandom.com）— 假设（Cloudflare 拦截）

- **信息架构**：MediaWiki 框架；顶部 **Fandom 全局巨导航**（游戏中心 / 宇宙切换），左侧 wiki 分类导航，主内容，右栏 Trending / Fandom / 广告。
- **视觉风格**：重 chrome，品牌感弱（Fandom 模板化），字体普通。
- **广告位**：**极 intrusive**——顶部 leaderboard、右栏、插页式 interstitial、自动播放视频、底部 sticky。
- **借鉴**：无。明确作为**反面教材**——其广告模式正是我们"广告与内容和平共处"要反着来的对象。

### 1.4 游民星空塞尔达专区（gamersky.com/z/zeldabotw2）— 实测

- **信息架构**：游戏"专区"页 = **左侧超长分类树**（主线 / 支线 / 武器防具 / 神庙 / 地图 / 图鉴 …，抓取可见完整树），主区为超长分页"图文攻略"（如抓取的 100+ 页 handbook），另有 tag 列表页（如 `tag.gamersky.com/handbook/...`）。
- **视觉风格**：门户风，密度极高，红色品牌点缀，信息堆砌。
- **广告位**：重——顶部 banner、右栏、文中、移动端，体验差（PRD 已记"广告重体验差"）。
- **借鉴**：① 中文、内容全；② 分类树覆盖广（我们复用其分类思路，但统一进 BOTW+TOTK 单站）。
- **规避**：① BOTW / TOTK **割裂**（我们统一站）；② 广告重；③ 检索弱（我们要强锚点 + 跨游戏检索）。

### 1.5 跨竞品结论

| 维度 | 可借鉴（吸收） | 反模式（禁止） |
|---|---|---|
| 导航 | 左分类树 + 右 TOC 双栏锚定（Game8 / ZD） | 首页变新闻流（ZD） |
| 内容 | 超全分类覆盖（游民）、结构化数据/SEO（Game8） | 双游戏割裂（游民）、模板化浅内容（Game8） |
| 广告 | — | 密集 + 插页 + 自动播放 + 不可关 sticky（Fandom / 游民） |
| 品牌 | 互动地图概念（ZD） | 重 chrome 模板感（Fandom） |

## 2. 对标品牌与设计语言【共享池写入】

### 2.1 候选对比（Notion / Stripe / Linear / IGN）

| 候选 | 适配度 | 理由 | 结论 |
|---|---|---|---|
| **Notion** | ★★★★★ | 块化内容、冷静 UI、顶级排版、左侧边栏 IA、超强可读性与组织力——完美命中"完整、好读、一眼找到"。双游戏层级 = Notion 的 workspace/page 树。 | **主对标** |
| **Stripe** | ★★★★☆ | 文档站（docs.stripe.com）清晰、可扫读的数据表 / 参数表——适合我们的武器/防具/素材数据库。 | 次级影响（数据表样式） |
| **Linear** | ★★★☆☆ | 深色、极快、微交互精致、键盘可达——适合"速度感/夜间模式"气质。 | 次级影响（微交互/快捷键） |
| **IGN** | ★★☆☆☆ | 游戏媒体巨头，内容密度高，但**广告 clutter 严重**——正是我们要避免的。 | **反面教材（广告勿学）** |

### 2.2 决策

- **主对标品牌 = Notion**：以"冷静的编辑型 wiki"为设计语言底座——低 chrome、高留白、内容优先、左侧分类树 + 面包屑 + 块化正文。
- **次级影响 = Stripe Docs（数据表）+ Linear（微交互/快捷键/深色气质）**：攻略中的"数据库页"（武器/防具/素材/图鉴）采用 Stripe 式清晰表格；全局支持 `Cmd/Ctrl+K` 检索与键盘可达，借鉴 Linear 的流畅感。
- **反面教材 = IGN / Fandom**：其广告 clutter 模式被明确列为禁止项，用于校准"广告与内容和平共处"的边界。

### 2.3 设计语言一句话

> **「Notion 的冷静组织力 + Stripe 的数据清晰度 + Linear 的流畅微交互」，做一个"塞尔达主题但去俗套"的编辑型攻略站——广告是原生的、可识别的、不阻断阅读的。**

## 3. 配色基调【共享池写入】

### 3.1 原则：融入塞尔达元素，但去俗套

- **俗套（禁止）**：亮 Link 绿 `#1E9E4A` + 土豪金 `#F5C518` 的渐变、卡通林克插画、霓虹光效。
- **我们的解法**：从海拉鲁的"自然 + 古老"中提取**去饱和、带温度**的色板——羊皮纸底（古 Hylian 经文感）、Hylian 鼠尾草绿（林克制服的"成年版"）、Triforce 黄铜（仅作极少量点缀，绝不做背景）。

### 3.2 色板（浅色 / 深色双主题）

| 语义 | 浅色 Token | 深色 Token | 说明 |
|---|---|---|---|
| 页面底 | `#FAF8F3` 羊皮纸白 | `#14171A` 洞窟炭 | 非纯白/纯黑，带温度 |
| 卡片面 | `#FFFFFF` | `#1B1F23` | — |
| 次级面 | `#F2EFE8` | `#1E2226` | 区块底色 |
| 主文字 | `#1B1D1A` | `#ECEEEA` | 暖近黑/暖近白 |
| 次文字 | `#5E615B` | `#A2A79E` | 带色调灰，非纯灰 |
| 弱文字 | `#9A9C95` | `#6E726B` | — |
| **主色** | `#2E6B4F` Hylian 绿 | `#4F9E78` | 链接/激活导航/主按钮 |
| 主色悬停 | `#265A42` | `#5FB089` | — |
| **点缀** | `#B68A3E` 黄铜 | `#C9A05A` | Triforce 金，**仅极少量** |
| 成功 | `#3F8F5B` | `#5BB57A` | 去饱和 |
| 警告 | `#C08A2E` | `#D6A23E` | — |
| 错误 | `#C2453E` | `#E0706A` | 去饱和红 |
| 边框 | `#E7E3DA` | `#2A2F33` | — |
| 强边框 | `#D8D3C8` | `#353B40` | — |
| 聚焦环 | `#2E6B4F` | `#4F9E78` | `:focus-visible` 2px |
| 广告底 | `#F7F5F0` | `#1E2226` | 与内容面微区分 |
| 广告标签 | `#9A9C95` | `#6E726B` | "广告"小字 |

> **红线自检**：无 `#7C3AED/#A855F7/#9333EA/#EC4899` 任一紫色；无纯黑 `#000`/纯灰 `#808080` 直出（均带温度）；主色为绿非紫。✅

### 3.3 塞尔达元素如何"克制"融入

- **Logo**：单色 Triforce（三个三角形几何标记），非金渐变。
- **分区图标**：Lucide 语义图标（剑=武器、盾=防具、地图=地图、火焰=神庙挑战…），**非 emoji**。
- **分隔**：细双线规则（呼应 Hylian 经文边框），不喧宾夺主。
- **点缀金**：仅用于"游戏切换激活态""稀有/付费标记""关键数据高亮"，**绝不做背景或渐变**。

## 4. 设计方向【共享池写入】

### 4.1 字体系统

```
--font-display: "Inter", "Noto Sans SC", -apple-system, BlinkMacSystemFont, sans-serif;
--font-body:    "Inter", "Noto Sans SC", -apple-system, BlinkMacSystemFont, sans-serif;
--font-mono:    "JetBrains Mono", "Fira Code", ui-monospace, monospace;
```

- **字号层级（7 级）**：12 / 14 / 16 / 18 / 20 / 24 / 32 / 40 px。
- **正文阅读**：详情页正文 17px、行高 1.75、内容列最大宽 720px（最佳阅读测量值）。
- **数据表**：`JetBrains Mono` 用于物品 ID、坐标（如 `(-1234, 0789, 0345)`）、配方比例。

### 4.2 视觉风格关键词

`冷静 · 编辑型 · 内容优先 · 低 chrome · 高留白 · 可扫读 · 双主题 · 流畅微交互`

### 4.3 全局信息架构（Shell）

```
┌─────────────────────────────────────────────────────────────┐
│ 顶栏(吸顶, 44px)                                              │
│ [Triforce]  切换[TOTK|BOTW]   🔍搜索(Cmd+K)   国内|海外  主题 │
├──────────┬──────────────────────────────────────┬──────────┤
│ 左栏(折叠)│ 面包屑：TOTK > 分类 > 标题            │ 右栏(详情)│
│ 分类树    │ ┌──────────────────────────────────┐ │ 本页目录  │
│ · 主线流程 │ │ 标题 + 元信息(更新/版本/时长)     │ │ (TOC锚点) │
│ · 神庙(152)│ ├──────────────────────────────────┤ │ 相关攻略  │
│ · 地图位置 │ │ 正文块：步骤/表格/提示/图集       │ │ [广告位]  │
│ · 武器装备 │ │                                  │ │          │
│ · 料理药剂 │ └──────────────────────────────────┘ │          │
│ · 任务     │                                      │          │
│ · 敌人Boss │                                      │          │
│ · 图鉴     │                                      │          │
│ · 技巧速通 │                                      │          │
└──────────┴──────────────────────────────────────┴──────────┘
       侧边 sticky [广告位]（可关闭，滚过首屏后出现）
```

- **顶栏**：游戏切换用分段控件（激活态黄铜描边），全局搜索 `Cmd/Ctrl+K` 命令面板，国内/海外按 locale 自动、可手动切。
- **左栏**：当前游戏的**分类树**（沿用游民/Game8 的超全分类，但统一进单站双游戏）。移动端收为抽屉。
- **右栏（仅详情页）**：本页 TOC（scroll-spy 高亮当前章节）+ 相关攻略 + 1 个赞助位。

### 4.4 首页（拒绝俗套 Hero 大图）

- **不**做"大标题 + 副标题 + 居中 CTA + 抽象 3D 图形"。
- **改为**：
  1. **快速入口带**（紧凑）：游戏切换 + 「大家都在搜」（真实搜索词驱动，如"防火女位置""全克洛格""料理配方"）+ 快捷芯片（神庙地图 / 全呀哈哈 / 防具图鉴）。
  2. **热门攻略**：真实攻略卡（封面为**真实游戏截图缩略图**，非抽象艺术），含标题/游戏标签/更新时间。
  3. **最新更新**：自动补抓/原创更新的时间线。
  4. **数据速查**：武器/防具速查小表（Stripe 式清晰表格）。
  5. 底部 1 条清晰标注「合作伙伴」的赞助条。
- 首页展示**真实产品内容**，不是口号。

### 4.5 列表 / 归档页

- 左树已全局固定；顶部增加**过滤芯片**（按区域 / 类型 / 难度）+ 排序（更新时间 / 热度）。
- 卡片网格或表格两种视图；分页或无限滚动。
- **原生广告**每 6 条注入 1 张卡（带「广告」标签 + 细边框，与内容卡同构但可识别）。

### 4.6 详情 / 攻略页

- 面包屑（游戏 > 分类 > 标题）。
- 标题 + 元信息（更新时间、来源原创/补抓、阅读时长、游戏版本号）。
- **右栏 TOC**：章节锚点 + scroll-spy 高亮；移动端收为顶部"目录"折叠。
- 正文块：导语 → 编号步骤 → 数据表 → 提示/警告 callout（Lucide 图标，**非 emoji**）→ 图集（真实截图）。
- 底部：上一篇/下一篇 + 相关攻略 + 文末广告位。
- **广告**：正文第 1 大段后 1 个原生卡 + 右栏 1 个赞助位。**绝不**插在句子中间打断阅读；**无**插页、**无**自动播放覆盖层。

### 4.7 广告位与内容和平共处（变现核心约束）

> 目标：广告是变现引擎（PRD 已确认 4 位），但**不冲突、不恶心**。借鉴 IGN/Fandom 的反面，立 7 条规矩：

1. **原生不破坏流**：广告做成与内容卡同构的组件，带明确「广告 / Sponsored」标签 + 细边框，读者一眼区分但不突兀。
2. **克制的数量与位置**（对齐 PRD 4 位）：
   - 顶部信息流：1 条原生 strip（首屏下方，非全屏拦截）。
   - 列表页：每 6 条 1 张原生卡。
   - 详情页：第 1 大段后 1 卡 + 右栏 1 位。
   - 地图 overlay：仅互动地图页（Backlog），静态图期用图片角标「广告」；不做覆盖式弹层。
3. **禁止的恶心广告**：全屏 interstitial、自动播放有声视频、悬浮覆盖层、伪装成内容的"暗广告"（必须标注）、不可关闭的底部 sticky。
4. **视觉分离但和谐**：广告卡用 `--ad-bg` + `--border-default` + 左上「广告」小字（`--ad-label`），**不抢主色**。
5. **性能与体验**：广告**懒加载**、占位尺寸固定（避免 CLS 布局抖动）、`async` 不阻塞正文、尊重 `prefers-reduced-motion`。
6. **双市场合规**：国内 = 合规联盟（百度联盟 / 腾讯广告，需备案资质）+ 自有赞助；海外 = Google AdSense / AdX，按 IP 切换；遵守 GDPR/CCPA + 国内个保法；提供「广告偏好」入口。
7. **可控可关**：侧栏 sticky 提供关闭按钮，且滚过首屏才出现；设置页可「减少广告干扰」（登录/会员降频）。
