# Spec - 塞尔达攻略网站 MVP v1.0.0

> 生成日期：2026-07-15
> 基于：PRD v0.1 + 架构设计文档 v1 + UIUX 文档 v1
> 状态：已确认（用户全权委托，Phase 1.5 自动生成）

---

## 1. 产品定义
- **一句话描述**：一个站搜遍塞尔达两代（TOTK+BOTW）的图文攻略站，中文百度、英文 Google 都能找到，原创打法 + 定时补抓兜底完整度，广告位天生为变现设计。
- **目标用户**：国内双坑收集党（核心变现）、海外英文玩家（AdSense）、新手路痴（流量入口）。
- **核心问题**：玩家到处搜攻略、攻略不完整、双游戏内容割裂。

## 2. MVP 范围（锁定）

| 优先级 | 功能 | 验收标准摘要 | RICE |
|--------|------|-------------|-------|
| P0 | SEO 基础设施 | 双 sitemap + hreflang + 结构化数据 + 自动提交，百度/Google 可收录 | 67.5 |
| P0 | 广告位系统 | 每页 3-4 位（信息流/侧边/文末/地图overlay），按 IP 地域切 AdSense/百度联盟 | 63.3 |
| P0 | 自动补抓管线 | 定时抓合规公开 wiki→转 Markdown→入库→重建 | 23.3 |
| P0 | 双游戏结构化内容库 | 每游戏 ≥ 核心流程 + Top 高频词，图文并茂（每页 ≥1 原创图） | 20.0 |
| P1 | 图文并茂呈现 | 每攻略页含原创插画/地图/SVG 物品图 + 结构化步骤 + FAQ | 纳入 P0 内容库 |
| P2 | 站内统一检索 | 跨 BOTW+TOTK 搜索（v1.1，Pagefind） | 36.0 |

### 明确不做（Won't Have）
- 自产视频攻略（原因：成本高、与文字攻略定位冲突）
- 重互动地图（原因：MVP 用 SVG 静态图替代）
- 登录/收藏/进度（原因：非变现核心）
- UGC 社区（原因：合规与维护成本）
- 超中英的多语言（原因：MVP 聚焦 zh/en）

## 3. 技术架构（锁定）
- **前端/交付形态**：零构建静态站（纯 HTML/CSS/JS），最终产物 = 静态 HTML（与 Astro SSG 产物一致，后期可无缝迁移 Astro）。
- **为何偏离 Astro 初始选型**：用户无开发/美工能力，零构建静态站可双击预览、拖拽部署，无需本地编译环境。架构师原选型仍是长期演进目标。
- **自动更新**：Node 爬虫脚本（`site/crawler/crawl.js`）+ GitHub Actions `schedule`（UTC）触发 → 写 Markdown/JSON 内容 → 提交 → 重新部署静态站。
- **广告方案**：`<AdSlot>` 组件统一注入；环境变量/配置切换 AdSense(`/en` 海外) 与百度联盟(`/zh` 国内)；未接广告前显示占位。
- **内容存储**：Markdown + frontmatter（git 管理），字段 `game`(totk|botw)、`lang`(zh|en)、`is_original`、`title`、`cover`。
- **部署**：Cloudflare Pages（全球）/ Netlify（拖拽）/ EdgeOne Pages（国内免备案）。

## 5. 设计 Token（锁定）
- **主色**：Hylian 绿 `#2E6B4F`
- **点缀**：Triforce 黄铜 `#B68A3E`
- **浅底**：羊皮纸 `#FAF8F3`；**深底**：洞窟炭 `#14171A`
- **字体**：系统无衬线 + 中文 `Noto Sans SC` 回退（无外链依赖，离线可用）
- **图标**：内联 SVG（禁 emoji、禁紫色渐变、禁俗套 Hero 大图）
- **主题**：浅 + 深双主题，localStorage 持久化
- **对标**：Notion（编辑型冷静 wiki），避开 IGN/Fandom 广告密集俗套
- **广告铁律**：原生 / 克制 / 禁插页 / 合规 / 可关 / 内容优先 / 不抢首屏

## 6. 自动更新机制（锁定）
- 触发：GitHub Actions `schedule`（如每日 02:00 UTC），亦可手动 `workflow_dispatch`。
- 流程：`crawl.js` 抓合规公开源（robots 允许、限流 ≥2s）→ 提取正文+配图引用 → 生成带 frontmatter 的 Markdown/JSON → `git commit` → 平台重建部署。
- 合规：仅公开可引用内容，保留 `source_url` 与抓取时间；原创度需 ≥60%（百度/AdSense 要求）。

## 7. 验收标准（锁定）

| 编号 | 功能 | Given | When | Then |
|------|------|-------|------|-------|
| AC1 | SEO 收录 | 发布 TOTK 攻略页 | 提交百度/Google | 72h 内被收录，FAQ 富媒体展示 |
| AC2 | 广告位切换 | 海外 IP 访问 | 加载页面 | 展示 AdSense 代码；国内 IP 展示百度联盟（占位亦可） |
| AC3 | 图文并茂 | 打开任意攻略页 | 渲染完成 | 含 ≥1 原创图 + 结构化步骤 + FAQ |
| AC4 | 双主题 | 点击主题切换 | 切换主题 | 深浅主题切换并持久化 |
| AC5 | 自动更新 | 到达定时触发点 | 爬虫执行 | 新内容入库生成新页面，旧内容更新时间刷新 |
| AC6 | 双市场 SEO | 英文用户经 Google | 抓取 | `hreflang=en` 被索引且与 `zh` 互为 alternates |

## 8. 边界与约束
- 不支持 IE；目标现代浏览器，ES2020+。
- 百度 `robots.txt` 的 sitemap 行**禁写 `sitemap-index.xml`**，须写 `sitemap-0.xml`；主动提交 + 自动推送。
- 广告变现非上线即得：AdSense 需 ≥30 原创 + 过审（过审率 <30%）；百度联盟需执照 + ICP 备案。MVP 验收不考核收入。
- 国内访问速度依赖托管/备案；未备案期走免备案节点或 BunnyCDN 港新，首屏 1-3s。
