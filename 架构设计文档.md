# 塞尔达攻略网站 · MVP 技术架构设计文档

> 作者：高见远（首席架构师）
> 日期：2026-07-14
> 范围：TOTK + BOTW 双游戏攻略站，原创 + 定时补抓公开 wiki，国内 + 海外双市场，目标快速广告变现、省成本、易部署、好收录。

---

## 0. 前提假设（按合理假设推进，未向用户确认）

| 编号 | 假设 | 影响 |
|------|------|------|
| A1 | MVP 为**内容驱动型静态站**（攻略/百科读多写少），无强实时交互 | 选 SSG 而非全栈 SSR，最省成本、最好收录 |
| A2 | "定时自动补抓"= 周期性抓取**公开、允许抓取的** wiki/攻略页，转 Markdown 入库后触发重建；非实时增量 | 爬虫放在 CI/cron，而非常驻服务 |
| A3 | 原创内容由**少数编辑/站长**维护，非多角色协同 | 采用 Git 式 Markdown + 可选可视化后台，不引入重 CMS |
| A4 | 双市场 = 同一套内容，**/zh 中文路径主投百度+国内联盟，/en 英文路径主投 Google+AdSense** | 单代码库、双子路径、双部署分发 |
| A5 | MVP 阶段**无 ICP 备案、无企业主体**，先用平台默认域名跑通；备案后绑自定义域名 | 国内访问走"免备案 CDN"或平台国内节点，速度非最优但可用 |
| A6 | 广告变现为**中期目标**：上线即申请，但过审需积累（内容量/流量/合规） | 架构预留广告位，但不把收入计入 MVP 验收 |

## 1. 技术选型对比矩阵

> 评分 1–5（5 最优）。维度权重见列下标。必覆盖方案：**Astro / Next.js SSG / Hugo** 三类，并附 1 个轻量对照（VitePress）。

| 维度（权重） | Astro（SSG） | Next.js（`output:export`） | Hugo（纯静态） | VitePress（轻量静态） |
|------|:--:|:--:|:--:|:--:|
| **SEO·百度/Google 收录**（高） | **5** | 4 | **5** | 4 |
| **自动更新管线·定时爬虫托管**（高） | **5** | 4 | 4 | 3 |
| **广告位集成·AdSense+国内联盟**（高） | **5** | 5 | 4 | 3 |
| **部署成本**（高） | **5** | 4 | **5** | **5** |
| **国内访问速度（依赖托管）**（高） | 4* | 2–3* | 4* | 4* |
| **内容管理·原创录入**（高） | **5** | 4 | 3 | 3 |
| **开发/维护成本·学习曲线**（中） | 4 | 3 | 3 | 4 |
| **综合加权** | **4.7** | 3.6 | 4.0 | 3.6 |

\* 国内访问速度主要取决于**托管平台**而非框架；静态产物可部署到任意国内 CDN，故框架侧给中性偏高分，具体见第 3、4 节托管方案。

### 1.1 各方案要点（基于官方文档）

**Astro（推荐）**
- 官方定位 "content-driven websites"，Server-First 渲染，零无用 JS，Core Web Vitals / SEO 友好。
- Content Collections（`glob`/`file` loader）+ Zod schema，适合结构化攻略内容（双游戏、双语言）。
- `@astrojs/sitemap` 官方集成，自动生成 `sitemap-0.xml` / `sitemap-index.xml`；SEO 元信息、JSON-LD 易注入。
- 任意 HTML/JS 注入 → 广告代码（AdSense / 百度联盟）可在 layout 级统一插入。
- 构建产物 `dist/`，纯静态，可部署到 Cloudflare Pages / Netlify / Vercel / 21YunBox / EdgeOne Pages。

**Next.js（`output: 'export'` 静态导出）**
- 官方静态导出：构建期生成 `out/` 全静态 HTML/CSS/JS，可托管任意静态服务器。
- 限制（官方明确）：`output:export` 下**不支持 ISR、`fallback:'blocking'`、服务端函数**；图片优化需自定义 loader；动态路由依赖 `getStaticPaths` 在构建期跑全量。
- 广告脚本注入方便，但 cron 触发重建需**外部调度**（Vercel Cron 或 GitHub Actions），因为静态导出无运行时 API。
- 默认托管 Vercel 在**国内访问慢**（DNS 污染/无大陆节点），需搭配国内 CDN。

**Hugo（纯静态生成器，Go）**
- 官方：每页 <1ms 生成，适合**大规模**站点（万级页面），原生 i18n、分类法、SEO 模板。
- 内容 = Markdown + frontmatter，Git 工作流天然契合爬虫入库；但无内置 Content Collections 式类型校验，需靠 archetypes/shortcodes。
- 广告代码靠 layout partials 注入，可行但交互组件（如联盟动态创意）开发体验弱。
- 部署到 21YunBox / Cloudflare Pages / 国内 OSS+CDN 均免费且快；**非技术编辑后台（Decap/Forestry）配置成本高于 Astro**。

**VitePress（轻量，Vue）**
- 适合文档型，SEO 与静态导出 OK，但攻略站需自定义列表/筛选/广告位，定制深度不如 Astro，内容集合能力弱，且仅当"极简文档站"才考虑。

## 2. 核心功能技术可行性验证

### 2.1 定时爬虫补抓（可行 ✅）
- **模式**：爬虫脚本（Node/Python）在调度器触发 → 抓取公开页（遵守 `robots.txt` 与版权、限流）→ 清洗为 Markdown + frontmatter（`game`、`source_url`、`fetched_at`、`is_original:false`）→ 写入内容目录 → git commit → 触发站点重建（deploy hook）。
- **调度托管对比**（已查证官方文档）：

| 方案 | 免费额度 | 最小间隔 | 时区 | 备注 |
|------|------|------|------|------|
| **GitHub Actions（schedule）** | 免费 | 5 分钟 | UTC（不可改） | 可延迟 5–15 分钟；仓库 60 天无活动自动停；**最适合跑爬虫+commit+deploy hook** |
| **Vercel Cron** | Hobby 仅 2 个/天 | Hobby 每天 1 次 | UTC | 触发 Serverless Function；Hobby 限制过严，且 Vercel 国内慢 |
| **Cloudflare Cron Triggers** | 免费（Workers 额度） | 1 分钟 | UTC | 边沿运行，调用 deploy hook 即可；需写 Worker 代码 |
| 外部 cron（cron-job.org / EasyCron） | 免费档够用 | 1 分钟 | 可设 | 仅触发 URL，不进仓库，适合"只调重建" |

- **结论**：MVP 用 **GitHub Actions** 跑爬虫（每日/每周一次足够），提交内容并触发托管平台 deploy hook 重建。多源/高频再升级 Cloudflare Cron Triggers。

### 2.2 SEO 双市场（可行 ✅，有注意事项）
- **通用**：静态全 HTML 是爬虫最友好形态（SSR/SSG 是 SEO 标配，纯 CSR 空壳会被百度/Google 判低质）。
- **Google**：`sitemap-index.xml` 直接提交 Search Console；1–3 天开始抓取。
- **百度（已查证坑）**：
  - 百度**不支持 `sitemap-index.xml`**，robots.txt 写索引会被判 "Blocked by Robots" → 必须**只写单 sitemap 路径**（如 `sitemap-0.xml`）或单独给百度提交简化 sitemap。
  - 新站**必须主动提交**（百度站长平台验证 + sitemap + 自动推送 JS），否则基本不被发现。
  - 中文内容在 `/zh` 路径，利于百度识别与排名；英文 `/en` 主投 Google。
- **结构化数据**：攻略页加 JSON-LD（`Article`/`HowTo`/`FAQ`），提升富摘要点击率。

### 2.3 广告位注入（可行 ✅，有过审门槛）
- 静态站可在 layout 任意位置放 `<AdSlot>` 组件（header / 文中插 / 侧栏 / footer），注入 AdSense 或百度联盟 JS。
- **AdSense（海外）**：需合规站（建议 ≥30 篇原创）、运营时长、内容政策（2024 起严打纯采集/低质 AI 内容，过审率 <30%）。
- **百度联盟（国内）**：需**企业/个体执照 + 域名 ICP 备案 + 接百度统计**；个人站仅限特定类目。
- **双轨**：`/zh` 注百度联盟（需备案域），`/en` 注 AdSense；用环境变量/路由切换广告代码，避免冲突。
- **关键提醒**：广告位是"先搭好、后过审"，**MVP 上线不等于立即有收入**（见不可行警告 §5）。

### 2.4 原创内容录入/管理（可行 ✅）
- **主方案：Markdown + frontmatter（Git-based）**，目录区分 `src/content/original/`（原创，人工写）与 `src/content/wiki/`（爬虫补抓，自动写）。
- **可视化后台（可选）**：接入 **Decap CMS**（开源、Git 后端），在 `/admin` 提供表单式编辑，提交即 commit 回仓库、触发重建；Astro 官方有完整集成指南。非技术编辑无需碰 Git。
- **重 CMS（Sanity/Contentful）不推荐 MVP**：引入外部依赖、配额与费用，违背"省成本、好部署"。

## 3. 选型结论与推荐架构

### 3.1 结论
**推荐：Astro（SSG）+ GitHub Actions 爬虫管线 + 双托管分发（Cloudflare Pages 全球 / EdgeOne Pages 或 21YunBox 国内）+ Markdown(Git) 内容 + Decap CMS 可选后台。**

推荐理由：
1. 内容驱动站最佳契合：零 JS 开销 → 加载快 → 百度/Google 收录友好、LCP 优。
2. Content Collections 天然支撑"双游戏 × 双语言 × 原创/补抓"的结构化内容。
3. 广告代码在 layout 级统一注入，双市场切换简单。
4. 纯静态产物 → 任意免费静态托管，国内/海外各选最优节点。
5. Decap CMS 让非技术编辑也能发文，且零额外费用。

### 3.2 推荐分层架构（ASCII）

```
                         ┌─────────────────────────────┐
                         │   调度层 (GitHub Actions)     │
                         │  cron: 每日 0 18 * * * (UTC)   │
                         │  1. 跑 crawler 脚本           │
                         │  2. 写 Markdown → git commit  │
                         │  3. 调 deploy hook 触发重建    │
                         └───────────────┬─────────────┘
                                         │
                         ┌───────────────▼─────────────┐
                         │   内容层 (Git Repository)     │
                         │  src/content/                 │
                         │   ├ original/{totk,botw}/     │  (人工/Decap 写)
                         │   └ wiki/{totk,botw}/         │  (爬虫自动写)
                         │  frontmatter: game,lang,...   │
                         └───────────────┬─────────────┘
                                         │  astro build
                         ┌───────────────▼─────────────┐
                         │   构建层 (Astro SSG)          │
                         │  Content Collections + Zod    │
                         │  @astrojs/sitemap             │
                         │  <AdSlot> 组件注入广告位        │
                         │  产物 → dist/ (纯静态)         │
                         └───────────────┬─────────────┘
                         ┌───────────────┴───────────────┐
             部署分发（同源产物，双节点）                   
        ┌──────────────────────┐      ┌──────────────────────┐
        │ 全球 / Google / AdSense│      │ 国内 / 百度 / 百度联盟   │
        │ Cloudflare Pages       │      │ EdgeOne Pages / 21YunBox│
        │ (或 Vercel)            │      │ (国内节点 CDN)         │
        │ DNSPod 智能解析分流      │      │ 默认域名先跑，备案后绑域  │
        └──────────┬───────────┘      └──────────┬───────────┘
                   │                              │
            ┌──────▼──────────────────────────────▼──────┐
            │  用户：/en 海外访客 → Cloudflare+AdSense       │
            │       /zh 国内访客 → 国内节点+百度联盟          │
            └─────────────────────────────────────────────┘
```

### 3.3 数据流（一次内容更新）
`wiki 源站` →(crawler, 守 robots/限流)→ `Markdown+frontmatter` → `git commit` →(GitHub Actions)→ `astro build` → `dist/` →(deploy hook)→ `Cloudflare/EdgeOne 重建` → 新页面被百度/Google 抓取。

### 3.4 技术栈清单
| 层 | 选型 |
|------|------|
| 框架 | Astro 最新稳定版（SSG，`output:'static'`） |
| 内容 | Markdown + frontmatter；Zod schema 校验 |
| 后台 | Decap CMS（`/admin`，可选） |
| SEO | `@astrojs/sitemap`、JSON-LD、百度自动推送 JS |
| 广告 | `<AdSlot>` 组件；AdSense(`/en`)、百度联盟(`/zh`) |
| 爬虫 | Node(`cheerio`/`tsx`) 或 Python(`requests`+`bs4`)，脚本化 |
| 调度 | GitHub Actions `schedule`（主）；Cloudflare Cron Triggers（升级） |
| 托管 | Cloudflare Pages（全球）+ EdgeOne Pages / 21YunBox（国内） |
| 分发 | DNSPod 智能解析按地域分流（备案后） |

## 4. 技术约束【共享池写入】

> 以下内容为团队共享决策，后续研发/测试/运维须遵守。

1. **浏览器支持**：不支持 IE；目标现代浏览器（Chromium / Firefox / Safari 近 2 版），ES2020+。
2. **构建产物**：Astro 输出纯静态 `dist/`（HTML/CSS/JS + sitemap + robots.txt）；不得依赖运行时服务端逻辑做内容渲染。
3. **渲染模式**：全站 SSG（`output:'static'`）；如需极少数动态接口（如 crawler 回调），单独用 Edge Function/Worker，不污染主站。
4. **定时任务托管**：爬虫与重建由 **GitHub Actions `schedule`** 驱动（UTC，最小间隔 5 分钟，可能延迟）；禁止依赖 Vercel Cron Hobby（每天仅 2 次，过严）。多源高频升级 Cloudflare Cron Triggers。
5. **内容边界**：`src/content/original/` 仅人工/Decap 写入；`src/content/wiki/` 仅爬虫写入；二者 frontmatter schema 必须含 `game`(totk|botw)、`lang`(zh|en)、`is_original`(bool)。
6. **双市场路由**：`/zh/**` 中文（百度 SEO + 百度联盟），`/en/**` 英文（Google SEO + AdSense）；同一内容双写两语言或按语言分别维护。
7. **百度 SEO 硬约束**：`robots.txt` 中 sitemap 行**不得写 `sitemap-index.xml`**，须写单文件 `sitemap-0.xml`；并到百度站长平台主动提交 + 自动推送。
8. **广告位规范**：广告代码统一走 `<AdSlot>`，禁止在页面散落脚本；`/zh` 与 `/en` 广告供应商经环境变量切换，禁止同页混用两家脚本。
9. **爬虫合规**：仅抓取**允许抓取（robots.txt 合规）且明确可引用的公开内容**；保留 `source_url` 与抓取时间；限流（建议 ≥2s/请求）；禁止突破版权/反爬。
10. **ICP 备案**：国内自定义域名投放百度联盟**必须 ICP 备案**（企业/个体执照）。MVP 未备案期用平台默认域名，百度联盟暂不接入，仅预留位。

## 5. 选型结论【共享池写入】

- **主栈**：Astro（SSG）。理由：内容驱动最优、SEO 满分、广告注入简单、静态产物任意免费托管、Decap CMS 零成本后台。
- **次选/备选**：Hugo（若页面量冲万级且编辑全为开发者，构建速度更极致）；Next.js SSG（若团队 React 重度依赖且能接受国内托管额外配置）。
- **否决**：纯 CSR（Vite/React SPA 直出）—— 百度/Google 收录差，直接排除；全栈 SSR 常驻服务 —— 成本与运维超 MVP 所需。
- **托管**：全球 Cloudflare Pages（或 Vercel），国内 EdgeOne Pages / 21YunBox；DNSPod 智能解析分流。
- **调度**：GitHub Actions 跑爬虫 + deploy hook 重建。
- **内容**：Markdown(Git) 为主，Decap CMS 为可选可视化后台。

## 6. 不可行警告【共享池写入】

1. **⚠ 广告变现非上线即得**：AdSense 需 ≥30 篇原创、运营时长、内容政策（严打低质 AI/采集，过审率 <30%）；百度联盟需**执照 + ICP 备案**。MVP 上线即有广告位，但**过审与产生收入需数周~数月积累**，不可把收入写进 MVP 验收或早期预期。
2. **⚠ 国内访问速度依赖托管/备案**：Vercel/Netlify 默认域名在大陆被污染/慢；即使 Cloudflare Pages 国内也非最优。未备案期只能走"免备案国内节点（EdgeOne/21YunBox/帽子云）"或海外+准国内 CDN（BunnyCDN 香港/新加坡节点），首屏可能 1–3s，**非极致快**。要最优需备案 + 国内 CDN，属后期项。
3. **⚠ 百度 `sitemap-index.xml` 不兼容**：若 robots.txt 误写索引文件，百度直接判封禁、零收录。必须按 §4-7 处理。
4. **⚠ 大规模站点构建瓶颈**：内容冲数万页时 `astro build` 全量重建变慢、GitHub Actions 时长受免费额度限制。缓解：按需构建（hybrid 模式延迟生成）、内容分仓、爬虫增量提交（仅变更页）。MVP 千级页面无碍。
5. **⚠ 爬虫法律/封禁风险**：自动补抓若违反源站 robots.txt 或版权、或限流不当，可能被封 IP / 侵权投诉。须 robots 合规 + 注明出处 + 限流 + 仅公开可引用内容；原创为主、补抓为辅的比例需保持健康（百度/AdSense 均要求原创度 ≥60%）。
6. **⚠ GitHub Actions 调度不确定性**：定时工作流 UTC、可能延迟 5–15 分钟、仓库 60 天无提交自动停用。对"每日补抓"可接受；若需精确分钟级，改用 Cloudflare Cron Triggers / 外部 cron。

## 7. 下一步建议（交付研发）
1. 初始化 Astro 项目，配置 `site`、双语言路由、`@astrojs/sitemap`、JSON-LD。
2. 定义 Content Collections schema（`original`/`wiki` × `totk`/`botw` × `zh`/`en`）。
3. 实现 `<AdSlot>` 与广告供应商环境变量切换。
4. 写 crawler 脚本 + GitHub Actions workflow（每日），对接 deploy hook。
5. 接入 Decap CMS（可选）；配置百度站长平台 + Search Console + 自动推送。
6. 部署：Cloudflare Pages（全球）+ EdgeOne Pages/21YunBox（国内）；DNSPod 分流（备案后）。
